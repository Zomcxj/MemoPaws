# SnapTrans 应用 Claude 风格重制计划

## 概述

将 SnapTrans（PySide6 桌面应用）的 5 个页面在保持布局不变的前提下，视觉风格迁移至 Claude 设计系统暗色模式。输出为 `.design` 画布项目（HTML + Tailwind CDN + Claude CSS 变量）。

## 当前状态分析

- **应用**: SnapTrans — 基于 PySide6 的截图 OCR 识别与翻译工具
- **当前风格**: 深色主题 + 暖橙色强调色（`#f4a261`），深灰/黑色背景（`#1a1a1a` ~ `#2d2d2d`）
- **5 个页面**: 设置页、贴图识别页、剪贴板历史页、备忘录页、密钥管理页
- **共享组件**: 左侧导航栏（5 个导航项）、卡片、按钮、输入框、标签切换、列表

## 目标风格：Claude 设计系统（暗色模式）

| 维度 | Claude Token 值 |
|------|----------------|
| 主背景 | `#262624` |
| 侧边栏 | `#1F1E1D` |
| 卡片/面板 | `#2C2C2B` |
| 主文字 | `#F1F1EF` |
| 次要文字 | `#B7B5A9` |
| 弱化文字 | `#8A8880` |
| 边框 | `#3E3E38` |
| 品牌强调色 | `#D97757`（赤陶色） |
| 成功色 | `#788C5D` |
| 错误色 | `#D64545` |
| 字体 | Poppins(UI) / Lora(阅读) / Newsreader(展示) / Geist Mono(代码) |
| 圆角 | 按钮 0px / 卡片 12px / 输入框 8px |
| 阴影 | 极微弱或无（偏好硬边分隔） |

### 页面生成计划（5 页并行）

每个页面严格保持截图中的布局结构，仅替换视觉样式：

1. **设置页** — 卡片式设置区域，映射 Claude token 到卡片、输入框、切换按钮
2. **贴图识别页** — 三栏布局 + 工具栏，最复杂页面，Claude 硬边按钮风格
3. **剪贴板历史页** — 列表式布局，时间戳用等宽字体
4. **备忘录页** — 三栏 + Markdown 编辑器，正文用 Lora 衬线体，代码用 Geist Mono
5. **密钥管理页** — 卡片网格 + 列表，状态徽章用语义色

### 核心样式映射

| 组件 | 变更要点 |
|------|---------|
| 按钮 | 橙色 → 赤陶色 `#D97757`，圆角 → 0px（硬边） |
| 卡片 | 背景 → `#2C2C2B`，加 `#353533` 细边框，移除阴影 |
| 输入框 | 背景 → `#262624`，边框 → `#3E3E38`，聚焦 → `#D97757` |
| 侧边栏 | 背景 → `#1F1E1D`，选中项 → 赤陶色左边框 + 淡底 |
| 标签切换 | 选中 → 赤陶色淡底 + 赤陶色文字 |
| 导航项 | 文字 → `#B7B5A9`，选中 → `#F1F1EF` |

### 字体策略

- UI 文字（导航、按钮、标签）: Poppins
- 阅读内容（备忘录正文、说明）: Lora
- 技术文本（时间戳、API URL、代码）: Geist Mono
- 展示标题: Newsreader

### 关键设计决策

- **暗色模式**: 使用 Claude 设计系统 `.dark` 区块的 token 值
- **强调色克制**: `#D97757` 仅用于侧边栏选中、主按钮、选中标签
- **硬边优先**: 按钮无圆角，卡片保留 12px 圆角
- **无阴影**: 用边框和背景色差异替代阴影
- **字体加载**: 通过 Google Fonts CDN 引入 4 个字体族

## 验证步骤

1. 运行 `scan-design-directory.mjs` 验证 `.design` 文件完整性
2. 检查 5 个页面均正确渲染
3. 确认所有颜色使用 Claude token 变量而非硬编码
4. 确认布局结构与截图一致（无布局变更）
